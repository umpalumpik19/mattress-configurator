# Инструкции по настройке административной панели

## Обзор

Я успешно реализовал полноценную административную панель для вашего сайта конфигуратора матрасов. Панель включает управление заказами, отслеживание статуса, автоматические уведомления по электронной почте и современный адаптивный пользовательский интерфейс.

## 🔧 Шаги по настройке

### 1. Миграция базы данных

**ВАЖНО: Вы должны выполнить эти SQL-команды в редакторе SQL Supabase перед использованием административной панели.**

Откройте файл `database-migrations.sql` и выполните все SQL-команды в панели управления Supabase:

1. Перейдите в панель управления вашего проекта Supabase
2. Перейдите в раздел "SQL Editor"
3. Скопируйте и вставьте весь контент файла `database-migrations.sql`
4. Нажмите "Run"

Это выполнит следующее:
- Добавит поля для управления статусами в таблицу `orders`
- Создаст историю отслеживания статуса заказов
- Добавит таблицу для аутентификации администраторов
- Создаст необходимые индексы и функции
- Настроит политики безопасности на уровне строк (Row Level Security)

### 2. Установка зависимостей

В проект добавлены следующие пакеты:

```bash
npm install react-router-dom bcryptjs jwt-decode
```

Эти пакеты уже установлены на основе обновлений в `package.json`.

### 3. Переменные окружения

Убедитесь, что переменные окружения Supabase правильно настроены в файле `.env`:

```
REACT_APP_SUPABASE_URL=ваш_supabase_url
REACT_APP_SUPABASE_ANON_KEY=ваш_supabase_anon_key
```

Для системы отправки писем убедитесь, что в окружении Supabase Edge Function установлены следующие переменные:

```
MAILERSEND_API_KEY=ваш_mailersend_api_key
MAILERSEND_FROM_EMAIL=ваш_email_отправителя
MAILERSEND_FROM_NAME=Конфигуратор матрасов
ADMIN_EMAIL=ваш_email_для_уведомлений_администратора
```

### 4. Развертывание обновленной Edge Function

Вам нужно развернуть обновленную функцию отправки писем в Supabase:

```bash
# Перейдите в корневую папку вашего проекта
cd ваш-директория-проекта

# Разверните обновленную функцию
supabase functions deploy send-email
```

## 📋 Возможности административной панели

### Панель управления
- **Статистика заказов**: Визуальный обзор всех статусов заказов
- **Недавние заказы**: Последние заказы с быстрым просмотром статуса
- **Карточки с обобщением**: Общее количество заказов, выручка, ожидающие заказы, средняя стоимость заказа

### Управление заказами
- **Полный список заказов**: Таблица с возможностью поиска и фильтрации
- **Фильтрация по статусу**: Просмотр заказов по определенному статусу
- **Фильтрация по способу доставки**: Разделение заказов на доставку и самовывоз
- **Фильтрация по датам**: Фильтрация по дате создания заказа
- **Поиск**: По имени клиента, email или ID заказа
- **Пагинация**: Эффективная работа с большим количеством заказов

### Рабочийස

#### Для заказов с доставкой:
1. **В обработке** → **Одобрено** → **Передано в доставку** → **Доставлено**
2. Любой статус может быть изменен на **Отменено**

#### Для заказов с самовывозом:
1. **В обработке** → **Одобрено** → **Готово к самовывозу**
2. Любой статус может быть изменен на **Отменено**

### Уведомления по электронной почте

Система автоматически отправляет письма клиентам, когда:

1. **Заказ создан**: "Ваш заказ находится в обработке"
2. **Одобрено**: Разные сообщения для доставки и самовывоза
3. **Передано в доставку**: Включает дату и временной интервал доставки
4. **Доставлено**: Подтверждение успешной доставки
5. **Готово к самовывозу**: Уведомление о готовности заказа к самовывозу
6. **Отменено**: Уведомление об отмене

### Действия администратора

Для каждого заказа администратор может:
- **Просмотреть детали**: Полная информация о заказе, включая товары, данные клиента, информацию о доставке
- **Обновить статус**: Изменение статуса заказа с модальными окнами подтверждения
- **Добавить заметки**: Внутренние заметки администратора для каждого изменения статуса
- **Установить детали доставки**: Дата и временные интервалы для заказов с доставкой
- **Просмотреть историю**: Полный аудит изменений статусов

## 🚨 Устранение неполадок

### Распространенные проблемы

1. **Административная панель не загружается**
   - Убедитесь, что `react-router-dom` установлен
   - Проверьте конфигурацию маршрутов в `AppRouter.js`
   - Проверьте консоль браузера на наличие ошибок

2. **Ошибки базы данных**
   - Убедитесь, что все SQL-миграции выполнены
   - Проверьте структуру таблиц в панели управления Supabase
   - Убедитесь, что политики RLS включены

3. **Письма не отправляются**
   - Проверьте конфигурацию ключа API MailerSend
   - Убедитесь, что Edge Function развернута
   - Проверьте журналы функций Supabase

4. **Проблемы с аутентификацией**
   - Убедитесь, что пользователь администратора существует в таблице `admin_users`
   - Проверьте localStorage на наличие состояния аутентификации
   - Очистите кэш браузера и localStorage

### Шаги по отладке
1. Проверьте консоль браузера на наличие ошибок JavaScript
2. Проверьте сетевые запросы в инструментах разработчика браузера
3. Проверьте журналы Supabase на наличие ошибок базы данных
4. Просмотрите журналы Edge Function для проблем с отправкой писем
